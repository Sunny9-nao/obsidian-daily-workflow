# AI プロンプト：アイデア深掘り・設計化（単一ノート運用）

アイデアをデイリーノートから拾い、同一ノートで設計まで育てるための AI 用プロンプト。既存の INDEX/ETL を壊さず、必要なときだけ `03.ideas/` にノートを作る。

---

## 使うテンプレートとルール
- テンプレート: `vault/03.ideas/Templates/Idea-Design.md`
- 1 ノートでアイデア〜設計まで扱う。階層分割なし。
- frontmatter 必須項目
  - `type: idea`（固定）
  - `stage: idea | design | done`（シンプルに 3 段階）
  - `src`: デイリーのブロック参照（例 `[[00.daily/2025-12-16#^20251216-i1]]`）
  - `owner`: 空でも可。分かれば入れる。
  - `last-updated`: ノートを更新した日付
  - `tags: [idea]`（必要に応じてタグ追加可、形式は `#prefix:value`）
- 本文は自由（見出しは必要に応じて追加/改名/並び替え OK）
  - 迷ったらテンプレの見出し（`Summary` / `Problem / Why` / `Notes` / `設計メモ` / `決定事項 / TODO`）を起点にする
  - 追加して良い例: `調査/参考`、`代替案`、`決定ログ`、`成功指標`、`実装メモ`、`オープンな論点` など
- stage の進め方: 初期は `idea`、設計を書き始めたら `design`、実装/検証まで完了したら `done`
- デイリーの INDEX 行や Block ID は変更しない。リンクのみ参照する。

---

## AI への指示（プロンプト本文としてそのまま使う想定）
以下を厳守してノートを生成・更新してください。

1. デイリーノートの日付とアイデアの Block ID（例 `^20251216-i1`）を受け取り、`src` を `[[00.daily/YYYY-MM-DD#^YYYYMMDD-iN]]` 形式で設定する。ID が不明なら質問して確定する。
2. ノートは `vault/03.ideas/<タイトル>.md` に作成/更新する。既存なら追記・上書き、無ければ新規。
3. テンプレート `Idea-Design.md` の frontmatter を満たすこと：
   - `stage` は `idea | design | done` のいずれか。設計を書き始めたら `design` に進める。
   - `last-updated` は更新日を入れる。
   - `tags` は `#prefix:value` 形式のみ。不要なら `tags: [idea]` のまま。
4. 本文の書き方：
   - テンプレの見出しは「おすすめ」であり固定ではない。必要に応じて `##` 見出しを自由に増やしてよい。
   - `Summary`: 1〜3 行で要約（迷ったら必ず書く）。
   - `Problem / Why`: 目的・課題・なぜ今か。
   - `Notes`: 関連情報、依存関係、参照リンク。デイリーの参照もここに置く。
   - `設計メモ`: 設計が必要になった時点で追記（箇条書きで良い）。
   - `決定事項 / TODO`: 決まったことと次アクション（チェックボックス可）。
   - 追加セクションは「情報が迷子にならない」命名にし、既存セクションの内容は勝手に削除しない。
5. 既存ノートを更新する場合は、破壊的変更を避け、追記・補正で進める。不要なフィールドは空のままでよい（消さない）。
6. 出力はノート全文のみ（余計な説明文を付けない）。コードブロックが必要なら ```` を使う。

---

## 使い方（人間側の手順メモ）
1. デイリーの `[idea] ... ^YYYYMMDD-iN` を決める。
2. AI に「日付・Block ID・タイトル・書きたい内容（メモでもOK）」を渡し、このプロンプトを貼る。
3. 初回は `stage: idea` で生成。設計を書き始めたら同じノートを `stage: design` に更新させる。
4. 進捗に合わせて追記依頼を出す（設計メモを埋める、決定事項を更新する、など）。done になったら `stage: done` に更新させる。

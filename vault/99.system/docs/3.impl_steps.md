# 実装ステップガイド（新人向け）

Dataview ベースのシンプル運用。「環境準備→初回作成→索引確認」までを手順化。

---

## 0. 事前前提

- macOS
- Obsidian がインストール済み
- VS Code + GitHub Copilot（編集用）

---

## 1. 環境準備（初回のみ）

### 1.1 Obsidian セットアップ

1. Obsidian を起動
2. 「Open folder as vault」で `vault/` フォルダを選択
3. 設定 → コミュニティプラグイン → セーフモードをオフ
4. 「Browse」から Dataview を検索してインストール
5. Dataview を有効化

### 1.2 Dataview 設定

1. 設定 → Dataview
2. 以下を確認/設定：
   - Enable JavaScript Queries: ON（任意）
   - Enable Inline Queries: ON

### 1.3 テンプレート確認

- `00.daily/Templates/Daily Note Simple.md` が存在することを確認

---

## 2. Day1 動作確認（最小ループ）

### 2.1 Daily Note を作成

1. `00.daily/` に新規ファイル作成（例：`2025-01-01.md`）
2. テンプレートをコピーまたは AI で生成
3. 各セクションに 1 項目以上記入

例：
```markdown
---
date: 2025-01-01
---

## Summary

初日の動作確認

## Learnings

- Dataview の基本を学んだ #ai

## Ideas

- ワークフロー改善のアイディア #workflow

## Actions

- [ ] 明日の計画を立てる

## Experiments

- 新しいツールを試した

## Notes

詳細メモをここに
```

### 2.2 索引を確認

1. Obsidian で `01.index/01.learnings.md` を開く
2. Dataview が動的に一覧を表示することを確認
3. `01.index/03.actions.md` で未完了アクションを確認

---

## 3. 日次運用（毎日）

### 3.1 Daily Note 作成

1. VS Code で `00.daily/YYYY-MM-DD.md` を新規作成
2. AI（GitHub Copilot）に自然文を渡して整形
3. セクション構成とタグを確認
4. 保存

### 3.2 索引確認

1. Obsidian で `01.index/` を開く
2. 各索引ページを確認
3. 必要に応じてタグパネルで絞り込み

---

## 4. アクション管理

### 4.1 未完了アクションの確認

1. `01.index/03.actions.md` を開く
2. 未完了のタスク一覧を確認

### 4.2 アクションを完了にする

1. 元の Daily Note を開く
2. `- [ ]` を `- [x]` に変更
3. 保存 → actions.md の一覧から自動的に消える

---

## 5. タグ運用

### 5.1 標準タグの確認

- `99.system/docs/tag-taxonomy.md` を参照
- 既存タグから選定することを優先

### 5.2 新しいタグの追加

1. tag-taxonomy.md に新しいタグを追加
2. Daily Note で使用開始

### 5.3 タグ棚卸し（週次推奨）

1. tag-taxonomy.md を確認
2. 類似タグを統合
3. 過去の Daily Note を更新（必要に応じて）

---

## 6. ブログ化フロー

### 6.1 ネタを発見

1. `01.index/02.ideas.md` を確認
2. タグで関連する学びを集約

### 6.2 下書き作成

1. `02.blog/drafts/` に新規ファイル作成
2. 関連する Learnings/Ideas をコピー
3. AI で記事化

---

## 7. トラブルシュート

| 症状 | 原因 | 対処 |
|------|------|------|
| 索引に表示されない | セクション名のスペルミス | `## Learnings` 等を正確に記述 |
| 索引に表示されない | Templates フォルダが含まれている | クエリに `!contains(file.path, "Templates")` を確認 |
| Dataview が動かない | プラグイン未有効 | 設定 → コミュニティプラグイン → Dataview を有効化 |
| タグで絞り込めない | タグ形式が不正 | `#tag` 形式で記述（空白不可） |

---

## 8. 次のステップ

- 索引をカスタマイズ（Dataview クエリを編集）
- テーマ別索引を追加（`01.index/` に新規ファイル）
- ブログ記事を公開（`02.blog/published/` へ移動）

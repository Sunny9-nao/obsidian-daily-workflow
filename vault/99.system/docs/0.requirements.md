# 要件定義

## 1. 背景

- 日次の振り返りやメモを続けたいが、「文章を整える/まとめる」コストが高く、継続が難しくなりがち
- 読書や記事の学びをブログ化したいが、「読み終えてから一気に書く」は負荷が高く、後回しになりやすい
- 日々のメモに埋もれがちなアイディアを、後から活用できる形で取り出したい

## 2. 目的

1. **AI 経由で日次ノートを整形**し、文章を考えるコストを減らして継続しやすくする
2. 日次ノートを「後でブログ化しやすい形」に残し、**日次メモ → ブログ下書き**へ流用できる仕組みを作る
3. アイディアを索引化し、好きなタイミングで取り出して「やってみる」につなげる

### 2.1 成功/品質基準（定量）

- 日次作成〜索引確認まで 1 分以内（Dataview で自動表示）
- ブログ下書き着手までのリードタイムを日次+3 日以内にできること
- オンボーディング：新規参加者が Day1 で「daily 作成 → Obsidian で索引確認」を 10 分以内で完了できること

## 3. 基本方針（設計原則）

- **Obsidian 中心の運用**：編集・閲覧・索引確認すべて Obsidian で完結
- **VS Code + AI で編集**：GitHub Copilot 等で Daily Note を効率的に作成
- **Markdown が一次情報（正）**：ブログ化に必要な文脈・長文は Markdown に残す
- **スクリプト不要**：Dataview プラグインで動的に索引を表示（ETL/DB 不要）
- **タグで分類**：標準化されたタグ体系で検索・集約を容易に
- **AI 出力の一貫性を担保する**：タグ体系を明文化し、AI がルールに従ってタグ付け

## 4. 技術選定（理由：簡潔）

- **Obsidian**：ローカルで Markdown を直接扱え、Dataview で動的索引、バックリンク/グラフで関連発見
- **Dataview プラグイン**：SQL ライクなクエリで動的に一覧表示、スクリプト不要
- **Markdown（Vault）**：一次情報としての可搬性が高く、将来のツール変更にも強い
- **VS Code + GitHub Copilot**：AI 支援で Daily Note を効率的に作成
- **タグ体系（tag-taxonomy.md）**：標準化されたタグで分類・検索を容易に

## 5. 対象データ（扱う情報）

- 日次の振り返り・ログ（Summary）
- 読書/記事メモ（Notes セクション）
- 学び（Learnings セクション）
- アイディア（Ideas セクション）
- 次の一手（Actions セクション：チェックボックス形式）
- 実験・試行（Experiments セクション）
- タグ（標準化された 20 種類程度）

## 6. 機能要件

### 6.1 日次ノート生成（AI 整形）

- ユーザーは自然言語で入力する
- AI は `00.daily/YYYY-MM-DD.md` を一定フォーマットで生成する
- セクション構成：Summary / Learnings / Ideas / Actions / Experiments / Notes
- タグは標準化されたリスト（tag-taxonomy.md）から選定

### 6.2 タグ運用（標準化）

- タグは `#value` 形式（シンプル）
- 標準タグリスト（約 20 種類）から選定
- カテゴリ：AI・自動化 / 技術・ツール / 学習 / テーマ / アウトプット / コンテキスト
- 新しいタグが必要な場合は tag-taxonomy.md に追加してから使用

### 6.3 索引化（Dataview 動的表示）

- Dataview クエリで各セクションから自動抽出
- 索引ページ：dashboard / learnings / ideas / actions / experiments
- タグで絞り込み可能

### 6.4 Obsidian での活用導線

- Dashboard で全体概要を確認
- 各索引ページで一覧表示
- タグパネルで分類別にアクセス
- バックリンク/グラフで関連発見

### 6.5 ブログ化支援

- Ideas/Learnings から記事ネタを発見
- タグで関連する学びを集約
- Daily Note の Notes セクションに詳細を記録

## 7. 非機能要件（運用・品質）

- **継続しやすさ**：日次作成は「自然文 → AI 整形」で完了すること
- **シンプルさ**：スクリプト実行不要、Obsidian のみで完結
- **ローカル完結**：Vault 内で閉じた運用
- **即時反映**：Dataview で保存即座に索引に反映

## 8. 前提条件

- macOS
- Obsidian（Dataview プラグイン必須）
- VS Code + GitHub Copilot（編集用）
- Vault の命名規則：`00.daily/YYYY-MM-DD.md`

## 9. 成果物

- 日次ノート（Markdown）：`00.daily/YYYY-MM-DD.md`
- 索引ページ（Dataview）：`01.index/*.md`
- ブログ下書き：`02.blog/drafts/*.md`
- タグ体系：`99.system/docs/tag-taxonomy.md`
- セットアップガイド：`99.system/docs/4.obsidian_setup.md`

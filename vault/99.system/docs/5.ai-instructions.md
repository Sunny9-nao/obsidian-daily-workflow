# AI 向け規約文書（Daily Note 生成用）

> このドキュメントは GitHub Copilot 等の AI が Daily Note を生成・修正する際に守るべき規約を定めています。
> Dataview による動的索引のため、セクション構成とタグ形式を厳守してください。

---

## 1. 基本原則

- **セクション構成を厳守**：Dataview がセクション名で抽出するため
- **タグは標準リストから選定**：tag-taxonomy.md を参照
- **Actions はチェックボックス形式**：`- [ ]` を使用
- **日本語と英語の混在 OK**：ただしタグは英語推奨

---

## 2. Daily Note の必須セクション構成

### 2.1 ファイル命名規則

```
00.daily/YYYY-MM-DD.md
```

例：`00.daily/2025-12-21.md`

### 2.2 Frontmatter

```yaml
---
date: YYYY-MM-DD
---
```

- date のみ必須
- シンプルに保つ

### 2.3 セクション構成（必須）

```markdown
---
date: 2025-12-21
---

## Summary

今日のハイライトを 1 行で

## Learnings

- 学んだこと 1 #tag1
- 学んだこと 2 #tag2

## Ideas

- [ ] アイディア 1 #tag
- [ ] アイディア 2

## Actions

- [ ] やること 1 #tag
- [ ] やること 2

## Experiments

### 🧪 実験タイトル

**目的**:

**結果**:

## Notes

詳細なメモをここに自由に記述
```

## 3. 各セクションの仕様

### 3.1 Summary

- 今日のハイライトを **1-2 行** で
- タグ不要
- 自由記述

### 3.2 Learnings

- **リスト形式**：`- ` で始める
- **タグ推奨**：末尾に `#tag` を付ける
- 複数タグ可：`#ai #reading`

```markdown
## Learnings

- MCP サーバーの仕組みを理解した #ai
- Playwright でブラウザ自動操作を学んだ #playwright #testing
```

### 3.3 Ideas

- **チェックボックス形式**：`- [ ] ` で始める（タスクとして管理）
- **タグ推奨**：関連技術やテーマ
- 未着手：`- [ ]`
- 着手済：`- [x]`

```markdown
## Ideas

- [ ] Playwright Agent で自動テスト生成 #ai #playwright
- [ ] 読書メモの自動要約 #ai #reading
```

### 3.4 Actions

- **チェックボックス形式**：`- [ ] ` で始める
- **未完了**：`- [ ]`
- **完了済**：`- [x]`
- タグは任意

```markdown
## Actions

- [ ] MCP サーバーを試す #ai
- [x] ドキュメントを読む #reading
```

### 3.5 Experiments

- **見出し形式**：`### 🧪 タイトル` で始める
- **目的と結果**を記載
- 試したこと・実験したこと

```markdown
## Experiments

### 🧪 Dataview クエリのパフォーマンステスト

**目的**: 大量データでの表示速度を確認

**結果**: 100 件程度なら問題なし
```

### 3.6 Notes

- **自由記述**
- 詳細なメモ、読書メモ、振り返りなど
- 長文 OK

## 4. タグ規約

### 4.1 タグ形式

- シンプル形式：`#value`
- 例：`#ai`、`#playwright`、`#reading`

### 4.2 標準タグリスト

tag-taxonomy.md で定義された約 20 種類から選定：

| カテゴリ     | タグ                                        |
| ------------ | ------------------------------------------- |
| AI・自動化   | #ai, #playwright, #cli                      |
| 技術・ツール | #ruby, #rails, #python, #git, #obsidian     |
| 学習         | #reading, #course                           |
| テーマ       | #productivity, #workflow, #testing, #design |
| アウトプット | #blog, #talk                                |
| コンテキスト | #work, #personal, #event                    |

### 4.3 タグ選定ルール

1. **既存タグを優先**：tag-taxonomy.md を確認
2. **類似タグを統合**：例）mcp, agent, copilot → `#ai`
3. **新規タグは事前に追加**：tag-taxonomy.md に追加してから使用

### 4.4 ❌ NG 例

- `#ai-agent`（ハイフン入り）→ `#ai` を使う
- `#mcp_server`（アンダースコア入り）→ `#ai` を使う
- `#AI`（大文字）→ `#ai` を使う

## 6. Dataview との連携

### 6.1 なぜセクション構成が重要か

- Dataview は `meta(L.section).subpath` でセクション名を判定
- セクション名が正確でないと抽出されない

### 6.2 注意点

- 見出しは正確に `## Learnings` 等と記述
- スペルミスに注意
- 余分な空白を入れない

---

## 7. チェックリスト（生成時に確認）

- [ ] Frontmatter に `date: YYYY-MM-DD` がある
- [ ] 6 つのセクションがすべて存在する（Summary, Learnings, Ideas, Actions, Experiments, Notes）
- [ ] Actions はチェックボックス形式（`- [ ]`）
- [ ] タグは標準リスト（tag-taxonomy.md）から選定
- [ ] タグは小文字・英語（`#ai`、`#playwright`）
